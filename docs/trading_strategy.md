# トレーディング戦略の解説

## 概要

本システムは **テクニカル分析 + AI判断** のハイブリッド戦略を採用している。
テクニカル指標で市場の状態を数値化し、Claude AI が複数指標を総合的に分析して売買を判断する。

---

## テクニカル指標

### RSI (Relative Strength Index)

**設定**: 14期間

相対力指数。直近の値動きから「買われすぎ / 売られすぎ」を判定する。

```
RSI = 100 - 100 / (1 + RS)
RS = 平均上昇幅 / 平均下落幅
```

| 値 | 解釈 | シグナル |
|----|------|---------|
| 0〜30 | 売られすぎ | 買いシグナル（反発期待） |
| 30〜70 | 中立 | 方向性なし |
| 70〜100 | 買われすぎ | 売りシグナル（反落期待） |

### MACD (Moving Average Convergence Divergence)

**設定**: 短期EMA=12, 長期EMA=26, シグナル線=9

移動平均収束拡散法。トレンドの方向と強さを示す。

```
MACD線 = 短期EMA - 長期EMA
シグナル線 = MACD線の9期間EMA
ヒストグラム = MACD線 - シグナル線
```

| 状態 | 解釈 | シグナル |
|------|------|---------|
| ヒストグラムが正転 | 上昇トレンド | 買いシグナル |
| ヒストグラムが負転 | 下降トレンド | 売りシグナル |
| ゼロ付近 | トレンド転換の可能性 | 要注意 |

### ボリンジャーバンド

**設定**: 20期間, 2σ

```
中央線 = 20期間の単純移動平均
上限 = 中央線 + 2 × 標準偏差
下限 = 中央線 - 2 × 標準偏差
BB位置 = (現在価格 - 下限) / (上限 - 下限)
```

| BB位置 | 解釈 | シグナル |
|--------|------|---------|
| 0以下 | 下限を下回った | 買いシグナル（バンド内への回帰期待） |
| 0〜0.3 | 下限付近 | 弱い買いシグナル |
| 0.3〜0.7 | バンド中央 | 中立 |
| 0.7〜1.0 | 上限付近 | 弱い売りシグナル |
| 1.0以上 | 上限を上回った | 売りシグナル |

### 補助指標

| 指標 | 計算方法 | 用途 |
|------|---------|------|
| 1時間変動率 | (現在価格 - 1時間前) / 1時間前 × 100 | 短期のモメンタム判定 |
| 4時間変動率 | (現在価格 - 4時間前) / 4時間前 × 100 | 中期のトレンド判定 |

## AI 分析の仕組み

### プロンプト構成

Claude AI に以下の情報を渡す:

```
1. システムプロンプト（トレーディングアナリストとしての役割と判断基準）
2. 対象通貨のテクニカル指標
   - 現在価格, RSI, MACD（線/シグナル/ヒストグラム）, BB（上限/中央/下限/位置）
   - 1時間/4時間変動率
   - 現在の保有量
3. 他通貨の参考情報（BTC下落時はアルト連動を考慮）
```

### AI の出力

```json
{
  "action": "BUY" | "SELL" | "HOLD",
  "confidence": 0.0 〜 1.0,
  "reason": "判断理由を1-2文で"
}
```

### 確信度フィルター

- **60%未満**: 実行しない（HOLD扱い）
- **60%以上**: 売買を実行

これにより、AIの判断が曖昧な場面での無駄な売買を抑制する。

### 応答バリデーション

AIの応答に不正がある場合:
- `action` が BUY/SELL/HOLD 以外 → HOLD に変更
- `confidence` が数値でない / 範囲外 → 0.0 に設定
- JSONパースエラー / タイムアウト → HOLD（確信度0.0）を返却

## 注文戦略

### 指値注文（デフォルト）

```
BUY の場合: 現在価格 - (現在価格 × LIMIT_OFFSET_PCT / 100)
SELL の場合: 現在価格 + (現在価格 × LIMIT_OFFSET_PCT / 100)
```

デフォルトの `LIMIT_OFFSET_PCT = 0.05` の場合:
- BTC = ¥15,000,000 → BUY指値 = ¥14,992,500（7,500円下）
- ETH = ¥280,000 → BUY指値 = ¥279,860（140円下）

**メリット:**
- Maker リベート（-0.01%）を獲得
- スリッページなし

**デメリット:**
- 約定しない可能性がある（次のサイクルで再判断）

### 成行注文

損切り時のみ使用。確実に約定させることを優先する。

## ポジション管理

### 新規購入

```
目標投入額 = 総資産 × max_position_ratio (10%)
購入サイズ = 目標投入額 / 現在価格
```

### 追加購入（買い増し）

目標比率に未達の場合のみ実行:

```
現在のポジション評価額 = 保有量 × 現在価格
総資産概算 = JPY残高 + ポジション評価額
目標額 = 総資産概算 × max_position_ratio
不足額 = 目標額 - ポジション評価額
追加購入サイズ = 不足額 / 現在価格
```

追加購入時、エントリー価格は加重平均で更新:

```
新しい平均価格 = (旧価格 × 旧数量 + 新価格 × 新数量) / (旧数量 + 新数量)
```

### 売却

AIがSELL判断を出した場合、保有する全量を売却する。

## リスク管理

### 損切り（ストップロス）

```
損切り条件: (現在価格 - エントリー価格) / エントリー価格 <= -stop_loss_pct
```

| 通貨 | 損切りライン |
|------|------------|
| BTC | -2% |
| ETH | -3% |
| XRP | -3% |
| SOL | -3% |

BTCはボラティリティが相対的に低いため、損切りラインを狭く設定。
アルトコインはBTCより変動が大きいため、余裕を持たせている。

損切りは **成行注文** で即座に実行。AI判断よりも優先される。

### 利確（テイクプロフィット）

```
利確条件: (現在価格 - エントリー価格) / エントリー価格 >= take_profit_pct
```

| 通貨 | 利確ライン |
|------|----------|
| BTC | +4% |
| ETH | +5% |
| XRP | +6% |
| SOL | +6% |

損切り幅の約2倍に設定（リスクリワード比 ≈ 2:1）。

### 資金配分

```
総資産: 100%
├── BTC: 最大 10%
├── ETH: 最大 10%
├── XRP: 最大 10%
├── SOL: 最大 10%
└── JPY（待機資金）: 60%以上
```

60%以上をJPYで保持することで:
- 急落時の追加購入余力を確保
- 全損リスクを大幅に低減
- 新たな投資機会に対応可能

## マルチ通貨戦略

### 通貨間の情報共有

各通貨のAI分析時に、他通貨のテクニカル指標（価格、RSI、変動率）を参考情報として渡す。

**意図:**
- BTC が急落している場合、アルトコインも連動して下落しやすい
- 市場全体のセンチメントを考慮した判断が可能

### 独立したポジション管理

各通貨は独立したポジションとして管理される:
- 損切り/利確のエントリー価格は通貨ごとに個別管理
- 1つの通貨の損切りが他の通貨に影響しない
- `risk_state.json` に全通貨の状態を一元的に保存

## 想定パフォーマンス

### 期待値の考え方

```
1回の取引あたり:
  利確時: +4% (BTC), +5% (ETH), +6% (XRP/SOL)
  損切時: -2% (BTC), -3% (ETH/XRP/SOL)
  リスクリワード比: 約 2:1

勝率50%と仮定した場合（投入額10,000円）:
  期待値 = 0.5 × 400 + 0.5 × (-200) = +100円/取引 (BTC)
  期待値 = 0.5 × 500 + 0.5 × (-300) = +100円/取引 (ETH)
```

> ただし、実際の勝率はAIの判断精度・市場環境に大きく依存する。
> 上記はあくまでリスクリワード比が適切であることの説明。

### 手数料の影響

```
指値（Maker）: -0.01%（リベート） → 10,000円の取引で +1円
成行（Taker）:  0.05%            → 10,000円の取引で -5円

年間推定取引回数: 4通貨 × 96回/日（15分間隔）× 365日 ≈ 140,000サイクル
  ※ 全サイクルで約定するわけではない
```

指値注文中心の運用により、手数料がプラスに働く点が本システムの強み。
