# Crypto Auto Trader

GMOコイン取引所と Claude AI を組み合わせた、マルチ通貨対応の暗号資産自動売買システム。

BTC / ETH / XRP / SOL の4銘柄を同時に運用し、テクニカル指標を基に Claude AI が売買を判断する。
指値注文（Maker）をデフォルトとし、GMOコインの Maker リベート（-0.01%）を活用する。
Webダッシュボードからリアルタイムで全通貨の状況を監視・制御できる。

## 特徴

- **マルチ通貨同時運用** — BTC, ETH, XRP, SOL を同時にトレード（通貨ごとに有効/無効切替可能）
- **AI 売買判断** — Claude AI がテクニカル指標 + 出来高を分析し、BUY / SELL / HOLD を判断
- **指値注文（Maker）** — デフォルトで指値注文を使用し、手数料リベートを獲得
- **トレーリングストップ** — 利確ライン突破後、最高値から一定%下落で売却（利益を最大化）
- **自動損切り** — AI判断とは独立して、設定したラインで成行注文により即執行
- **追加購入（ナンピン）** — 目標比率未達の場合、加重平均で追加購入
- **LINE 通知** — 約定・損切り・利確・エラー時にLINEにプッシュ通知
- **取引履歴の永続化** — SQLiteに全取引を記録。累計損益・勝率・日次損益グラフを表示
- **パフォーマンスダッシュボード** — 勝率・累計損益・最大利益/損失をリアルタイム表示
- **初回起動時の自動復元** — GMO約定履歴から平均取得単価を自動算出
- **Web ダッシュボード** — ブラウザから全通貨のモニタリング・制御が可能
- **エントリー価格永続化** — 再起動しても損益管理が途切れない
- **Dry Run モード** — 実際の注文を出さずに動作確認（実残高は表示される）
- **CLI 分析モード** — Claude Code CLI を使えばAPI利用料が不要（サブスク内で利用可能）
- **AWS EC2 対応** — デプロイスクリプト・systemdサービス同梱で24/365稼働

## 対応通貨

| シンボル | 銘柄 | 最小注文 | 損切り | 利確トリガー | トレーリング | 特徴 |
|---------|------|---------|--------|-------------|-------------|------|
| **BTC** | ビットコイン | 0.00001 | -2% | +4% | 最高値-2% | 基軸通貨、流動性最大 |
| **ETH** | イーサリアム | 0.001 | -3% | +5% | 最高値-2% | 時価総額2位、DeFi基盤 |
| **XRP** | リップル | 1 | -3% | +6% | 最高値-3% | 国内人気高、流動性高 |
| **SOL** | ソラナ | 0.01 | -3% | +6% | 最高値-3% | 高速チェーン、成長著しい |

各通貨は総資産の **10%** を上限に独立運用される（最大4通貨で40%投入、60%はJPYで待機）。

設定は [`currencies.json`](currencies.json) で通貨ごとにカスタマイズ可能。

## 動作フロー

```
15分ごとのサイクル
│
├─ 1. 全通貨のローソク足を一括取得（GMOコイン klines API）
│     └─ 取得失敗時は CryptoCompare API にフォールバック
│
├─ 2. テクニカル指標を計算
│     ├─ RSI (14期間)
│     ├─ MACD (12/26/9)
│     ├─ ボリンジャーバンド (20期間, 2σ)
│     └─ 出来高比率（現在出来高 / 20期間平均）
│
├─ 3. 通貨ごとに以下を実行:
│     │
│     ├─ 未約定の指値注文があれば状態確認
│     │   ├─ 約定済み → ポジション更新、DB記録、LINE通知
│     │   ├─ 未約定 → キャンセルして次のサイクルで再判断
│     │   └─ 部分約定 → 約定分を反映
│     │
│     ├─ ポジション保有中なら最高値を更新 → 損切り/トレーリングストップ判定
│     │   ├─ 損切りライン到達 → 成行で即売却 → LINE通知
│     │   └─ トレーリングストップ発動 → 指値で売却 → LINE通知
│     │
│     ├─ Claude AI にテクニカル指標 + 出来高 + 他通貨の状況を送信
│     │   └─ BUY / SELL / HOLD + 確信度 + 理由 を受信
│     │
│     └─ 確信度 60% 以上なら売買を実行
│         ├─ BUY: 目標比率未達なら購入（追加購入も可）→ LINE通知
│         └─ SELL: 全ポジションを売却 → LINE通知
│
└─ 4. 15分間待機 → 次のサイクルへ
```

## 設定リファレンス

### グローバル設定 (`src/config.py`)

| 項目 | デフォルト | 説明 |
|------|-----------|------|
| `TRADE_INTERVAL_SEC` | 900（15分） | 分析・売買サイクルの間隔 |
| `CANDLE_PERIOD_SEC` | 3600（1時間） | ローソク足の時間枠 |
| `CANDLE_COUNT` | 100 | 取得するローソク足の本数 |
| `ORDER_TYPE` | `LIMIT` | 注文方式（`LIMIT` / `MARKET`） |
| `LIMIT_OFFSET_PCT` | 0.05 | 指値のオフセット（現在価格からのずれ%） |
| `MIN_TRADE_JPY` | 1,000 | 最小取引額（円）。これ未満の注文は出さない |

### テクニカル指標パラメータ (`src/config.py`)

| 項目 | デフォルト | 説明 |
|------|-----------|------|
| `RSI_PERIOD` | 14 | RSI の計算期間 |
| `MACD_FAST` | 12 | MACD 短期EMA |
| `MACD_SLOW` | 26 | MACD 長期EMA |
| `MACD_SIGNAL` | 9 | MACD シグナル線 |
| `BB_PERIOD` | 20 | ボリンジャーバンド期間 |
| `BB_STD` | 2 | ボリンジャーバンド標準偏差倍率 |

### 通貨別設定 (`currencies.json`)

| 項目 | 説明 |
|------|------|
| `enabled` | `true` で自動売買を有効化 |
| `name` | 日本語表示名 |
| `min_order_size` | GMOコインの最小注文単位 |
| `size_decimals` | 注文数量の小数桁数 |
| `max_position_ratio` | 総資産に対するこの通貨の目標投入比率 |
| `stop_loss_pct` | 損切りライン（エントリー価格からの下落率） |
| `take_profit_pct` | トレーリングストップの発動条件（この上昇率を超えたら追従開始） |
| `trailing_stop_pct` | トレーリングストップ幅（最高値からの下落率で売却） |

## リスク管理の仕組み

### ポジションサイズ制御

- 各通貨は総資産の **10%** を上限に運用（`max_position_ratio`）
- 4通貨合計で最大 **40%** をポジションに投入、**60%** はJPYで待機
- 最小取引額（1,000円）未満の注文はスキップ

### 追加購入（買い増し）

目標比率（10%）に未達の場合、AIがBUY判断を出せば差分を追加購入する。

```
例: 総資産 10万円、BTC目標 = 1万円分
  現保有: 3,000円分 → 差分 7,000円分を追加購入
  平均取得単価は加重平均で自動計算
```

### 損切り

エントリー価格から `stop_loss_pct` 以上下落したら **成行注文で即売却**。AI判断より優先される。

### トレーリングストップ（利確）

従来の固定利確に代わり、トレーリングストップで利益を最大化する。

```
仕組み:
1. エントリー後、15分ごとに最高値を記録
2. 最高値がエントリー + take_profit_pct を超えたらトレーリング有効化
3. 最高値から trailing_stop_pct 下落したら売却

例: BTC（take_profit=4%, trailing_stop=2%）
  ¥100で購入 → ¥104で利確ライン突破 → トレーリング開始
  → ¥110まで上昇（最高値更新）
  → ¥107.8（最高値から-2%）で売却 → 利益 +7.8%

  固定利確なら +4% で終わるところを +7.8% 獲得
```

### 出来高分析

AIは出来高データも考慮して判断する。

- **出来高比率 > 1.5**: トレンドの確認材料（信頼度が高い）
- **出来高比率 < 0.5**: ダマシの可能性（慎重に判断）

### 指値注文のライフサイクル

```
指値注文送信 → 次のサイクル（15分後）で状態確認
  ├─ 約定済み → ポジション更新、DB記録、LINE通知
  ├─ 未約定 → キャンセル → 次のサイクルで再判断
  ├─ 部分約定 → 約定分を反映、残りはキャンセル
  └─ 確認エラー → 3回リトライ → キャンセル試行後クリア
```

未約定の注文がある間は新規注文を出さないため、連続購入は発生しない。

### エントリー価格の永続化

`risk_state.json` に通貨ごとのエントリー価格・保有量・最高値を保存。
アプリを再起動しても損益管理が途切れない。

初回起動時にポジションがあるがエントリー価格が不明な場合は、GMOコインの約定履歴から平均取得単価を自動算出する。

## LINE通知

約定・損切り・利確・エラー時にLINEにプッシュ通知を送信する。

### 通知されるイベント

| イベント | 通知内容 |
|---------|---------|
| 約定（BUY/SELL） | 通貨、数量、価格、損益（SELL時） |
| 損切り発動 | 通貨、エントリー価格、現在価格、損失率 |
| トレーリングストップ発動 | 通貨、エントリー価格、現在価格、利益率 |
| エラー | エラー内容、発生時刻 |

### LINE Messaging API のセットアップ

1. [LINE Developers](https://developers.line.biz/) でアカウント作成
2. 「新規プロバイダー作成」→「Messaging API チャネル作成」
3. チャネル基本設定 →「チャネルアクセストークン（長期）」を発行
4. LINE アプリで作成したボットを友だち追加
5. `.env` に `LINE_CHANNEL_ACCESS_TOKEN` と `LINE_USER_ID` を設定

LINE設定が未入力の場合、通知は無視される（エラーにはならない）。

## パフォーマンスダッシュボード

全取引はSQLite（`trades.db`）に永続化され、再起動しても失われない。

### ダッシュボードの統計

- **累計損益** — SELL取引の合計損益（円）
- **勝率** — 利益が出たSELL / 全SELL
- **勝/負** — 勝ちトレード数 / 負けトレード数
- **平均損益%** — SELL取引の平均損益率
- **最大利益** — 単一取引の最大利益（円）
- **累計損益グラフ** — 日次推移をChart.jsで可視化

### API エンドポイント

| エンドポイント | 説明 |
|--------------|------|
| `GET /api/stats` | 全体統計（勝率、累計損益など） |
| `GET /api/stats?symbol=BTC` | 通貨別統計 |
| `GET /api/trades` | 取引履歴（直近100件） |
| `GET /api/trades?symbol=ETH&limit=50` | 通貨・件数指定 |
| `GET /api/daily_pnl` | 日次損益（直近30日） |
| `GET /api/daily_pnl?days=90&symbol=BTC` | 期間・通貨指定 |

## 損失シミュレーション

10万円を運用した場合の最悪シナリオ分析。

### 1回の全通貨同時損切り

| 通貨 | 投入額 | 損切り% | 損失額 |
|------|--------|---------|--------|
| BTC | ¥10,000 | -2% | -¥200 |
| ETH | ¥10,000 | -3% | -¥300 |
| XRP | ¥10,000 | -3% | -¥300 |
| SOL | ¥10,000 | -3% | -¥300 |
| **合計** | **¥40,000** | | **-¥1,100** |

1回の最大損失: **約 -1,100円（-1.1%）**

### 資産が半減（5万円）になるまで

```
毎回全通貨が同時損切りされる最悪ケースで:
  1回あたり約 -1.1% → 0.989^n = 0.5
  n ≈ 63回（全4通貨同時の損切りが63回連続）
  15分サイクルなら約 15.75 時間
```

現実には利確も混ざるため、この最悪ケースが実現する確率は極めて低い。

### 注意すべきリスク

| リスク | 説明 |
|--------|------|
| **スリッページ** | 急落時に損切りの成行注文が設定値より不利な価格で約定する |
| **API障害** | 取引所のAPI障害で注文が通らない/状態確認ができない |
| **スリープ** | PCがスリープすると損切りが発動しない（AWS推奨） |
| **相関リスク** | BTC急落時にアルト全体が連動下落し、4通貨同時に損切りされる |

## トラブルシューティング

### ローソク足が不足するエラー

```
[BTC] ローソク足データが不足 (28本)
```

GMOコインのklines APIは当日分のデータが少ない場合があります。複数日分を自動取得するため、初回起動時は数時間待てば十分なデータが蓄積されます。

### Claude CLI の認証エラー

```
Claude CLI 応答なし (code=1): Invalid API key
```

`.env` に `ANTHROPIC_API_KEY` が設定されていると、CLI モードと競合します。CLI モードでは自動的に `ANTHROPIC_API_KEY` を環境から除外しますが、解消しない場合は `.env` から当該行を削除してください。

### XRP の注文エラー (ERR-5114)

```
Decimal digits of size is invalid
```

XRPは注文数量に小数を含められません。`currencies.json` の `size_decimals` が `0` であることを確認してください。

### 指値注文が約定しない

指値注文は現在価格から `LIMIT_OFFSET_PCT`（デフォルト0.05%）だけ有利な価格で出します。価格変動が小さい場合は約定しないことがありますが、次のサイクル（15分後）で自動キャンセル → 再判断されます。

### PCスリープ時の挙動

PCがスリープすると全処理が停止し、損切りが発動しません。LIVE モードで運用する場合は **AWS EC2 へのデプロイを推奨** します。

ローカルPCで運用する場合: Windows 設定 → システム → 電源とスリープ → 「なし」に設定

### ポジションがあるのにBUYが出る

目標比率（10%）に未達の場合は追加購入が実行されます。これは正常な動作です。ログに「目標比率未達のため追加購入」と表示されます。

### EC2でIPアドレスが変わった

Elastic IPを設定していない場合、EC2の再起動でIPが変わります。AWSコンソール → EC2 → Elastic IP でIPを固定してください（インスタンス稼働中は無料）。
